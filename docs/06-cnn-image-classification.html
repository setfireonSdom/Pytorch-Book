7  Chapter 6 Understanding Image Models Through CNN Code

This chapter centers on code/06_cnn_classifier.py.

If the previous chapter was your first complete classification project, this one is your first project that treats images like images.

7.1 Start with the Overall Structure

The file still contains the same major pieces:

  1. the model class SimpleCNN
  2. the validation function evaluate()
  3. the dataset and DataLoader
  4. the training loop in main()

That is worth noticing.

It tells you something important:

when the task type stays the same, the broad structure of a PyTorch project often stays stable. The biggest changes usually happen inside the model.

7.2 Read SimpleCNN in Parts

self.features = nn.Sequential(
    nn.Conv2d(1, 16, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
    nn.Conv2d(16, 32, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
)

This part is the feature extractor.

Try not to memorize terminology yet. Read it as a sequence of shape changes.

7.2.1 Layer 1: nn.Conv2d(1, 16, kernel_size=3, padding=1)

The most important parts here are:

  • the input channel count is 1
  • the output channel count is 16

For FashionMNIST, the input shape is usually [batch_size, 1, 28, 28].

After this convolution, the shape becomes:

  • [batch_size, 16, 28, 28]

Why do height and width stay 28 x 28?

Because padding=1 keeps the spatial size unchanged for this kernel size.

7.2.2 Layer 2: nn.MaxPool2d(2)

This layer does not change the number of channels, but it cuts the spatial size in half.

So the shape becomes:

  • [batch_size, 16, 14, 14]

7.2.3 The Next Convolution and Pooling Layers

After the second convolution and pooling steps, the shape becomes:

  1. [batch_size, 32, 14, 14]
  2. [batch_size, 32, 7, 7]

This is the habit worth building:

after every major layer, ask yourself what the shape is now.

7.3 Why Is the Classifier Written This Way?

self.classifier = nn.Sequential(
    nn.Flatten(),
    nn.Linear(32 * 7 * 7, 10),
)

This part turns extracted features into class scores.

7.3.1 nn.Flatten()

The convolutional block outputs a 4D tensor.

The fully connected layer expects a 2D tensor, so we flatten first.

7.3.2 nn.Linear(32 * 7 * 7, 10)

This is not an arbitrary number.

It comes directly from the shape calculation above:

  • channels: 32
  • height and width: 7 x 7

So each image becomes 32 * 7 * 7 features before classification.

If you get this number wrong, the model usually fails with a dimension mismatch.

7.4 What Does forward() Show You?

def forward(self, x: torch.Tensor) -> torch.Tensor:
    x = self.features(x)
    return self.classifier(x)

These two lines are a clear example of how CNN code is often organized:

  1. first extract features
  2. then classify

You will see this pattern again in many more advanced vision models.

7.5 Notice What Did Not Change

for images, labels in train_loader:
    images = images.to(device)
    labels = labels.to(device)

    optimizer.zero_grad()
    logits = model(images)
    loss = criterion(logits, labels)
    loss.backward()
    optimizer.step()

This is almost the same training template as the previous chapter.

That is one of the most useful long-term lessons in PyTorch:

what transfers best is not memorizing one specific model, but learning which parts are stable and which parts can be swapped out.

7.6 The Main Ideas to Take Away

  1. CNNs suit image tasks because they preserve local spatial structure.
  2. Convolution layers often change the number of channels.
  3. Pooling layers often reduce height and width.
  4. The input size of a fully connected layer must come from shape reasoning, not guessing.
  5. The model may change, but the training loop often stays largely the same.

7.7 Useful Changes to Try

  1. Change the first convolution from 16 output channels to 32
  2. Add another convolution layer to features
  3. Recompute the input size of the final Linear layer yourself
  4. Temporarily print x.shape inside forward() and compare it with your manual reasoning
