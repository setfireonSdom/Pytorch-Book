4  Chapter 3 Learning the Training Loop Through Linear Regression

This chapter focuses on a single file: code/03_linear_regression.py.

It may look simple, but it is one of the most important files in the book.

Why? Because almost every more advanced task later on still relies on the same training structure.

4.1 First, Look at the Structure of the Code

This file has three main parts:

  1. make_dataset(): creates training data
  2. train_one_epoch(): wraps one epoch of training
  3. main(): builds the model, loss, and optimizer, then runs training

That is the basic shape of a minimal but usable training project.

4.2 Start with make_dataset()

features = torch.randn(num_samples, 1)
noise = 0.2 * torch.randn(num_samples, 1)
targets = 3.5 * features + 1.2 + noise
return TensorDataset(features, targets)

This code creates a dataset where there really is a pattern between input and target.

You can read it like this:

  • features are the inputs
  • targets are the true answers
  • the underlying rule is roughly y = 3.5x + 1.2
  • some noise is added to make it more realistic

The point of this setup is that we do not have to think about messy real-world data yet. We can focus on the training process itself.

4.3 Then Look at the Setup in main()

dataset = make_dataset()
loader = DataLoader(dataset, batch_size=32, shuffle=True)

model = nn.Linear(1, 1)
criterion = nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)

These lines assemble the four essential pieces of a training task:

  1. data
  2. model
  3. loss function
  4. optimizer

You will do this again and again in your own projects.

4.4 Why Linear Regression Is the Right Starting Point

It works so well for learning because:

  • the data is simple
  • the model is simple
  • the loss is simple
  • the training behavior is easy to interpret

That lets you focus on the loop itself instead of being distracted by model complexity.

4.5 Now Read train_one_epoch() Line by Line

This is the part you should keep coming back to:

for inputs, targets in loader:
    optimizer.zero_grad()
    predictions = model(inputs)
    loss = criterion(predictions, targets)
    loss.backward()
    optimizer.step()

Whenever you read a PyTorch training script, your eyes should learn to find these steps almost immediately.

4.5.1 for inputs, targets in loader:

Training does not happen on the whole dataset at once. It happens one batch at a time.

Why?

  • it uses memory more efficiently
  • it trains faster
  • it works better with GPUs

4.5.2 optimizer.zero_grad()

This clears the old gradients.

If you forget this step, gradients accumulate, and training usually behaves incorrectly.

4.5.3 predictions = model(inputs)

This is the forward pass.

The model uses its current parameters to make predictions for this batch.

4.5.4 loss = criterion(predictions, targets)

This compares predictions to targets and produces a scalar loss.

You can think of it as “how wrong is the model on this batch right now?”

4.5.5 loss.backward()

This computes gradients for the model parameters.

4.5.6 optimizer.step()

This is the step that actually updates the parameters.

Beginners often blur backward() and step() together. Keep the distinction clear:

  • backward() computes gradients
  • step() uses those gradients to change parameters

4.6 The Training Template Hidden in the Code

This loop is really just six actions:

  1. get a batch
  2. clear old gradients
  3. run the forward pass
  4. compute the loss
  5. run backpropagation
  6. update the parameters

If you can see those six actions clearly, more advanced code becomes much easier to read.

4.7 Finally, Look at the Epoch Loop

for epoch in range(1, 21):
    avg_loss = train_one_epoch(model, loader, criterion, optimizer)
    print(f"epoch={epoch:02d} loss={avg_loss:.4f}")

An epoch means one full pass through the training set.

At this point, two ideas should become clear:

  1. a batch is the basic unit of training work
  2. multiple epochs are needed for the model to improve gradually

4.8 What You Are Really Learning Here

This chapter is not really about linear regression.

It is about learning to see almost any training script as a variation on the same core pattern.

Once that pattern becomes familiar, CNNs, transfer learning, and even much larger models start to feel less mysterious.

4.9 Three Experiments to Try Right Away

  1. Change the learning rate from 0.1 to 1.0 and see whether training becomes unstable.
  2. Change the learning rate to 0.001 and see whether convergence becomes much slower.
  3. Change the batch size and watch whether the loss becomes smoother or noisier.
